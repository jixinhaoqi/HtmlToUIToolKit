# HTML 转 UI Toolkit 工具使用教程

[(English)](HTML转UIToolKit工具使用教程_EN.md)

本文档介绍如何使用 [`HTML转UIToolKit工具.html`](HTML转UIToolKit工具.html) 将 HTML/CSS 源码转换为 Unity UI Toolkit 的 UXML/USS 格式。

---

## 打开工具

有两种方式打开转换工具：

1. 在 Unity 编辑器菜单栏选择 **Tools > HtmlToUIToolKit > 浏览器打开工具**
2. 直接在文件管理器中双击打开 [`HTML转UIToolKit工具.html`](HTML转UIToolKit工具.html)

工具将在默认浏览器中打开，界面分为三个主要面板。

---

## 界面布局

```
+--------------------------------------------------------------+
|  [执行转换]   [下载 UXML]   [下载 USS]    状态消息             |
+--------------------------------------------------------------+
+------------------+------------------------+------------------+
|    HTML 源码     |      实时预览           |   UXML/USS 输出   |
|   （左侧面板）    |      （中间面板）        |   （右侧面板）     |
|                  |                        |                  |
|  粘贴 HTML 代码  |  iframe 渲染效果        |  上部: USS 代码   |
|                  |  可调整预览分辨率       |  下部: UXML 代码  |
|                  |                        |                  |
+------------------+------------------------+------------------+
```

### 各区域说明

| 区域 | 功能 |
| --- | --- |
| HTML 源码（左） | 粘贴或编辑 HTML/CSS 代码的输入区 |
| 实时预览（中） | 通过 iframe 渲染 HTML，支持调整预览分辨率（宽/高） |
| USS 输出（右上） | 转换后的 USS 样式表，语法高亮显示 |
| UXML 输出（右下） | 转换后的 UXML 结构，语法高亮显示 |
| 顶部按钮栏 | 执行转换、下载文件、复制代码、状态提示 |

---

## 操作步骤

### 第一步：粘贴 HTML 代码

将完整的 HTML 代码粘贴到左侧编辑区。建议包含完整的 `<html>`、`<head>` 和 `<body>` 标签，以确保浏览器正确渲染样式。

> 工具打开时会自动加载一个登录表单的示例 HTML，供参考。

### 第二步：调整预览分辨率

在中间面板上方的输入框中设置预览分辨率：

- **宽（默认 1920）**：预览视口宽度（像素）
- **高（默认 1080）**：预览视口高度（像素）

分辨率设置影响 CSS 像素值的计算精度。建议设置为接近实际展示场景（如游戏分辨率）的值。对于移动端 UI，可设置为 750x1334 或 1125x2436。

### 第三步：执行转换

点击顶部 **"执行转换"** 按钮（或直接修改 HTML 代码，工具会在 300 毫秒后自动触发转换）。

转换过程中，工具会：
1. 在 iframe 中渲染 HTML
2. 通过浏览器 DOM API 获取计算后的样式
3. 将 HTML 元素映射为 UI Toolkit 元素
4. 将 CSS 属性映射为 USS 属性
5. 生成格式化的 USS 和 UXML 输出

### 第四步：查看输出

右侧面板显示转换结果：

- **上部 USS**：包含所有 CSS 规则的 USS 样式表，使用类选择器
- **下部 UXML**：包含完整 UI 元素树的 UXML 结构，每个元素自动分配 `name` 属性

两个输出面板均支持语法高亮，方便阅读和检查。

### 第五步：导出文件

使用顶部按钮导出结果：

| 按钮 | 功能 |
| --- | --- |
| **下载 UXML** | 下载 `.uxml` 文件 |
| **下载 USS** | 下载 `.uss` 文件 |
| **复制**（每个输出面板右上角） | 复制对应代码到剪贴板 |

> 建议将下载的文件放入 Unity 项目的 Assets 目录下，通过 UI Builder 或 PanelSettings 引用。

## 按钮说明

| 按钮 | 位置 | 功能 |
| --- | --- | --- |
| 执行转换 | 顶部 | 手动触发 HTML → UXML/USS 转换 |

## 注意事项

### 浏览器兼容性

工具使用 ES6 语法和 DOM API，需要现代浏览器支持（Chrome 80+、Edge 80+、Firefox 75+、Safari 13+）。

### 分辨率设置建议

- **编辑器工具 UI**：1920x1080
- **手机竖屏 UI**：750x1334
- **平板 UI**：1536x2048
- **自定义**：按实际展示场景设置

### 样式限制

参考 [`HtmlToUIToolKit.md`](../Documentation/HtmlToUIToolKit.md) 中的 CSS 属性映射参考和已知限制章节。

### 路径后处理

生成的 UXML/USS 中图片引用使用相对路径。如需使用 Sprite Atlas，请在 Unity 中右键文件使用 **HtmlToUIToolKit** 菜单进行路径转换。

## 常见问题

### Q: 转换后样式与预览不一致？

A: 部分 CSS 属性在 Unity UI Toolkit 中没有完全对等的实现（如渐变背景会转为纯色、Grid 会转为 Flexbox）。建议在转换后通过 UI Builder 手动微调。

### Q: 图片没有显示？

A: 检查图片路径是否正确，以及图片是否已导入 Unity。可使用 **Assets > HtmlToUIToolKit** 菜单进行路径后处理。

### Q: 某些元素被识别为错误的 UI Toolkit 类型？

A: 工具基于 HTML 标签和计算样式进行类型推断。可通过修改 HTML 标签来调整映射结果（如将 `<div>` 改为 `<button>` 可显式映射为 Button）。
